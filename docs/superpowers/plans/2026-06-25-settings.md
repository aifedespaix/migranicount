# Settings (Theme / Dyslexia Font / Migraine Mode) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add manual theme switching (Clair/Sombre/Auto/Migraine), a dyslexia-friendly font choice (Lexend/OpenDyslexic), and a research-backed migraine color mode to Migracount's Settings screen.

**Architecture:** A new Pinia store (`useSettingsStore`) owns theme/font preference state, persists it via the existing `getJSON`/`setJSON` localStorage helpers, and applies it to the DOM via `data-theme`/`data-font` attributes on `<html>`. `theme.css` switches from `@media (prefers-color-scheme: dark)` to explicit `[data-theme="..."]` selectors (light implicit default, dark, migraine). `chartTheme.ts` is fixed to react to the store instead of the raw media query, closing a pre-existing gap where chart colors never updated on a theme change. Two font families are self-hosted as local `.woff2` files for the offline-first PWA.

**Tech Stack:** Vue 3 `<script setup>`, Pinia (setup-store syntax, matching `src/stores/declencheurs.ts`/`medocsFavoris.ts`), Vitest + jsdom, existing `src/storage/storage.ts` (`getJSON`/`setJSON`), existing `src/styles/form.css` pill classes.

- [ ] **Step 3: Implement the store**

Create `src/stores/settings.ts`:

```ts
import { defineStore } from 'pinia'
import { ref, computed, watchEffect } from 'vue'
import { getJSON, setJSON } from '../storage/storage'

export type ThemeChoice = 'light' | 'dark' | 'auto' | 'migraine'
export type FontChoice = 'none' | 'lexend' | 'opendyslexic'

interface SettingsState {
  theme: ThemeChoice
  dyslexicFont: FontChoice
}

const STORAGE_KEY = 'settings'
const DEFAULTS: SettingsState = { theme: 'auto', dyslexicFont: 'none' }

export const useSettingsStore = defineStore('settings', () => {
  const initial = getJSON<SettingsState>(STORAGE_KEY, DEFAULTS)
  const theme = ref<ThemeChoice>(initial.theme)
  const dyslexicFont = ref<FontChoice>(initial.dyslexicFont)

  const systemPrefersDark = ref(
    typeof window !== 'undefined' && window.matchMedia
      ? window.matchMedia('(prefers-color-scheme: dark)').matches
      : false
  )

  if (typeof window !== 'undefined' && window.matchMedia) {
    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')
    mediaQuery.addEventListener('change', (e) => {
      systemPrefersDark.value = e.matches
    })
  }

  const resolvedTheme = computed<'light' | 'dark' | 'migraine'>(() =>
    theme.value === 'auto' ? (systemPrefersDark.value ? 'dark' : 'light') : theme.value
  )

  watchEffect(() => {
    if (typeof document === 'undefined') return
    document.documentElement.setAttribute('data-theme', resolvedTheme.value)
    if (dyslexicFont.value === 'none') {
      document.documentElement.removeAttribute('data-font')
    } else {
      document.documentElement.setAttribute('data-font', dyslexicFont.value)
    }
  })

  function persist(): void {
    setJSON(STORAGE_KEY, { theme: theme.value, dyslexicFont: dyslexicFont.value })
  }

  function setTheme(t: ThemeChoice): void {
    theme.value = t
    persist()
  }

  function setDyslexicFont(f: FontChoice): void {
    dyslexicFont.value = f
    persist()
  }

  return { theme, dyslexicFont, resolvedTheme, setTheme, setDyslexicFont }
})
```

### Task 3: Fix chart color reactivity to follow the settings store

**Files:**
- Modify: `src/utils/chartTheme.ts`

**Interfaces:**
- Consumes: `useSettingsStore().resolvedTheme` (`ComputedRef<'light'|'dark'|'migraine'>`) from `src/stores/settings.ts` (Task 1).
- Produces: `useChartThemeColors()` — same public signature as before (`{ accent: ComputedRef<string>, text: ComputedRef<string>, muted: ComputedRef<string> }`), no changes needed in any of its 3 consumers (`FrequencyChart.vue`, `IntensityChart.vue`, `EfficacyChart.vue`).

**Why this task exists:** `useChartThemeColors()` currently only re-reads CSS variables when the OS-level `prefers-color-scheme` media query fires. Before this plan, that was the *only* way the theme could change, so it happened to work. Task 1 introduces a manual override (`light`/`dark`/`migraine` set independent of the OS), which this matanism would silently miss — charts would keep stale colors after a manual theme switch. This task closes that gap.

- [ ] **Step 1: Replace the matchMedia listener with a store watch**

Replace the full contents of `src/utils/chartTheme.ts` with:

```ts
import { computed, onScopeDispose, ref, watch } from 'vue'
import { useSettingsStore } from '../stores/settings'

/**
 * Adds an alpha channel to a CSS color string.
 * Supports hex (#rrggbb) colors; falls back to wrapping with rgba via color-mix
 * for any other valid CSS color value.
 */
export function withAlpha(color: string, alpha: number): string {
  const trimmed = color.trim()
  const hexMatch = /^#([0-9a-fA-F]{6})$/.exec(trimmed)
  if (hexMatch) {
    const r = parseInt(hexMatch[1].slice(0, 2), 16)
    const g = parseInt(hexMatch[1].slice(2, 4), 16)
    const b = parseInt(hexMatch[1].slice(4, 6), 16)
    return `rgba(${r}, ${g}, ${b}, ${alpha})`
  }
  return `color-mix(in srgb, ${trimmed} ${Math.round(alpha * 100)}%, transparent)`
}

function readCssVar(name: string): string {
  return getComputedStyle(document.documentElement).getPropertyValue(name).trim()
}

/**
 * Reactive theme colors sourced from the active theme's CSS custom properties.
 * Re-evaluates whenever the settings store's resolved theme changes (manual
 * switch between light/dark/migraine, or a live OS-level scheme change while
 * the store is set to "auto").
 */
export function useChartThemeColors() {
  const settings = useSettingsStore()
  const themeVersion = ref(0)

  const stopWatch = watch(
    () => settings.resolvedTheme,
    () => {
      themeVersion.value++
    },
    { flush: 'post' }
  )

  onScopeDispose(() => {
    stopWatch()
  })

  const accent = computed(() => {
    void themeVersion.value
    return readCssVar('--color-accent') || '#8b5cf6'
  })

  const text = computed(() => {
    void themeVersion.value
    return readCssVar('--color-text') || '#2a2333'
  })

  const muted = computed(() => {
    void themeVersion.value
    return readCssVar('--color-muted') || '#6b6275'
  })

  return { accent, text, muted }
}
```

The `{ flush: 'post' }` option guarantees this watcher's callback runs after the settings store's own `watchEffect` (Task 1, which writes the `data-theme` attribute) has completed for the current reactivity flush, so `readCssVar` always sees the just-updated attribute — this holds regardless of which component instantiates the store first.

import { ref } from 'vue'
import { exportAll, importAll } from '../storage/migraineRepository'
import { useSettingsStore } from '../stores/settings'

const settings = useSettingsStore()
const confirming = ref(false)
const pendingJson = ref<string | null>(null)

function doExport() {
  const json = exportAll()
  const blob = new Blob([json], { type: 'application/json' })
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = `migracount-export-${new Date().toISOString().slice(0, 10)}.json`
  a.click()
  URL.revokeObjectURL(url)
}

function onFileSelected(e: Event) {
  const file = (e.target as HTMLInputElement).files?.[0]
  if (!file) return
  const reader = new FileReader()
  reader.onload = () => {
    pendingJson.value = reader.result as string
    confirming.value = true
  }
  reader.readAsText(file)
}

function confirmImport() {
  if (!pendingJson.value) return
  importAll(pendingJson.value)
  location.reload()
}
